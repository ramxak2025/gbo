# Как получить готовое приложение iBorcuha

Приложение **уже настроено на https://iborcuha.ru** — ничего редактировать не нужно. Вам осталось только его **собрать и установить на телефон**.

Есть **два пути**. Выберите один — любой работает.

---

## Путь 1 (самый быстрый, 5 минут) — через Expo Go

**Подходит** чтобы дать посмотреть приложение себе или бета-тестерам на Android/iOS **без публикации в сторы и без сборки APK**.

### На телефоне
1. Установите приложение **Expo Go** из Google Play (Android) или App Store (iOS).

### На компьютере
```bash
cd mobile
npm install
npx expo start --tunnel
```

В терминале появится **QR-код**. Откройте его:
- **Android** — в приложении Expo Go нажмите «Scan QR code».
- **iOS** — откройте камеру, наведите на QR — предложит открыть в Expo Go.

Приложение загрузится и будет работать с https://iborcuha.ru. Компьютер должен оставаться включён с запущенным `expo start`, пока пользователь тестирует.

## Путь 2 (настоящий APK-файл) — через EAS Build

**Подходит** если нужен самостоятельный `.apk` файл, который можно раздавать людям, не имея Expo Go.

### Шаг 1. Зарегистрироваться в Expo (бесплатно, 1 минута)
Откройте https://expo.dev и создайте аккаунт (e-mail + пароль).

### Шаг 2. На компьютере (нужен Node.js ≥ 18)
```bash
cd mobile
npm install -g eas-cli
npx eas-cli login              # введёте логин/пароль от expo.dev
npx eas-cli build --platform android --profile preview
```

EAS задаст 1–2 вопроса (создать проект? — **да**, сгенерировать keystore? — **да**).

### Шаг 3. Ждите 10–15 минут
APK собирается **в облаке Expo** — ваш компьютер не нагружается.

Когда сборка закончится:
- в терминале появится ссылка вида `https://expo.dev/artifacts/eas/xxxxx.apk`;
- и письмо на e-mail со ссылкой;
- и страница проекта на https://expo.dev со всеми сборками.

### Шаг 4. Установить на Android
1. Скачайте APK на телефон (открыв ссылку в браузере телефона, либо перекиньте файл).
2. При открытии файла Android спросит разрешение **«Установка из неизвестных источников»** — разрешите.
3. Нажмите «Установить». Готово.

## iOS (iPhone/iPad)

Если нужна сборка для iOS, команда та же, но платформа другая:

```bash
npx eas-cli build --platform ios --profile production
```

Отличия от Android:
- **Нужен Apple Developer аккаунт** ($99/год) — без него сборка только для симулятора.
- EAS попросит Apple ID, создаст провиженинг-профили и сертификаты автоматически.
- Результат — `.ipa` файл, который устанавливается через **TestFlight** (для тест-юзеров) или публикуется в App Store.

Пошагово — https://docs.expo.dev/submit/ios/

## Что уже настроено в приложении

- ✅ API-адрес: `https://iborcuha.ru/api`
- ✅ Deep links: `iborcuha://` + универсальные ссылки с `https://iborcuha.ru/*` открываются в приложении
- ✅ Push-уведомления (Expo Push) — токены регистрируются на бэке автоматически
- ✅ Иконка и splash-screen (из `mobile/assets/`)
- ✅ Package name: `com.iborcuha.app` (и для Android, и для iOS bundle id)
- ✅ Подхват токена из SecureStore — повторный вход не требуется
- ✅ Обновление токена (`/auth/refresh`) при истечении — прозрачно для пользователя

## Если сборка падает

1. `cd mobile && rm -rf node_modules && npm install` — чистая установка.
2. `npx expo-doctor` — покажет проблемы (версии пакетов).
3. `npx expo install --fix` — авто-фикс версий под текущий SDK.
4. Если ошибка на EAS — нажмите «View build details» → там подробный лог.

## Обновления приложения

После каждого изменения кода:
- **Через EAS Update (мгновенно, без пересборки APK)** — `npx eas-cli update --branch production` (требует настройки `expo-updates`).
- **Новая версия APK** — `npx eas-cli build --platform android --profile preview`, поднимите `version` в `app.json`.

---

**Итого: у вас в руках полностью готовое приложение. Осталось выбрать путь (Expo Go на 5 минут или EAS APK на 20 минут) и запустить 2-3 команды.**
