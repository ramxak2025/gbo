## 0. ЗАДАЧА

Создать мобильное приложение для iOS и Android, которое:
- работает с существующим бэкендом `https://iborcuha.ru/api`
- **визуально и функционально идентично** веб-версии на https://iborcuha.ru
- собирается в APK (Android) и IPA (iOS) через **Expo EAS Build**
- поддерживает 3 роли: **superadmin** (главный админ), **trainer** (тренер), **student** (спортсмен)
- поддерживает 16 пользовательских экранов (тренер + ученик) из 21 в вебе (5 admin-only экранов остаются только в вебе по RBAC)
- язык интерфейса: **русский**

### 4.1. Auth flow

1. При старте приложения `App.tsx` вызывает `useAuthStore.hydrate()`.
2. `hydrate`:
   - читает токен из SecureStore (ключ `iborcuha_token`)
   - если нет — state `unauthenticated`
   - если есть — читает user/student/role из SecureStore и ставит `authenticated`
   - если токен есть но user нет — делает запрос `GET /auth/me` для восстановления
3. Login:
   ```
   POST /auth/login { phone, password }
   → { token, userId, role, user, student?, studentId? }
   ```
   Сохраняем `token` в SecureStore, остальное в Zustand + SecureStore.
4. Logout:
   - `POST /auth/logout` (если доступен)
   - очистить SecureStore (`iborcuha_token`, `iborcuha_user`, `iborcuha_student`, `iborcuha_role`)
   - state `unauthenticated`
5. Refresh (прозрачно для UI):
   - `apiClient` ловит 401
   - single-flight вызов `POST /auth/refresh` → новый token
   - повторяет исходный запрос
   - если refresh сам вернул 401 → logout

### 4.2. Data flow

- `useDataBundle()` — `useQuery(['data'], () => api.get<DataBundle>('/data'))`
- Все мутации (create/update/delete) на `onSuccess` вызывают `queryClient.invalidateQueries({ queryKey: ['data'] })`.
- `staleTime: 30_000`, `gcTime: 5 * 60_000`.
- Retry: не ретраить 4xx, ретраить сеть до 3 раз с экспоненциальной задержкой.

## 5. API-КОНТРАКТ (бэкенд уже существует и работает — НЕ меняй его)

Base URL: `https://iborcuha.ru/api`
Auth: `Authorization: Bearer <JWT>` + HttpOnly cookie `token`

### 5.1. Health
- `GET /health` → `{ status, uptime, database, memory }` (без авторизации)

### 5.2. Auth
- `POST /auth/login` — body `{ phone: string, password: string }` → `{ token, userId, role, user, student?, studentId? }`, 401 при неверных данных, 429 при rate-limit
- `POST /auth/register` — body `{ name, phone, password, clubName?, sportType?, city?, consent: true }` → `{ ok: true }` (заявка на одобрение)
- `POST /auth/logout` → `{ ok: true }`
- `GET /auth/me` → `{ user, student?, role, studentId? }` или `null`
- `POST /auth/refresh` → `{ token }` (новый 30-дневный)

### 5.3. Снапшот данных
- `GET /data` → `DataBundle` (см. §5.12) — **главный источник** всех списков. Фильтрация по роли на бэке.

### 5.4. Ученики (students)
- `POST /data/students` — body: `StudentCreateInput` → `Student`
- `PUT /data/students/:id` — body: `StudentUpdateInput` → `Student`
- `DELETE /data/students/:id` → `{ ok: true }`

### 5.5. Группы (groups)
- `POST /data/groups` — `GroupCreateInput` → `Group`
- `PUT /data/groups/:id` — `GroupUpdateInput` → `Group`
- `DELETE /data/groups/:id` → `{ ok: true }`

### 5.6. Транзакции (cash)
- `POST /data/transactions` — `TransactionCreateInput` → `Transaction`
- `PUT /data/transactions/:id` → `Transaction`
- `DELETE /data/transactions/:id` → `{ ok: true }`

### 5.7. Посещаемость (attendance)
- `POST /data/attendance` — `{ groupId, studentId, date, present }` → `Attendance` (upsert)
- `POST /data/attendance/bulk` — `{ groupId, date, records: [{ studentId, present }] }` → `{ ok: true }`
- `DELETE /data/attendance` — `{ groupId, studentId, date }` → `{ ok: true }`

### 5.8. Турниры (tournaments)
- `POST /data/tournaments` — `{ title, date, location?, description?, coverImage? }` → `Tournament`
- `PUT /data/tournaments/:id` → `Tournament`
- `DELETE /data/tournaments/:id` → `{ ok: true }`
- `POST /data/tournament-registrations` — `{ tournamentId, studentId }` → `{ ok: true }` (idempotent)
- `DELETE /data/tournament-registrations` — `{ tournamentId, studentId }` → `{ ok: true }`

### 5.9. Внутренние турниры (internal tournaments, brackets)
- `POST /data/internal-tournaments` — `{ title, date?, brackets?, sportType?, coverImage? }` → `InternalTournament`
- `PUT /data/internal-tournaments/:id` → `InternalTournament`
- `DELETE /data/internal-tournaments/:id` → `{ ok: true }`

### 5.10. Материалы (materials)
- `POST /data/materials` — `{ title, description?, videoUrl, groupIds?, category?, customThumb? }` → `Material`
- `PUT /data/materials/:id` → `Material`
- `DELETE /data/materials/:id` → `{ ok: true }`

### 5.11. News, Author, Push
- `POST /data/news` — `{ title, content?, groupId? }` → `News`
- `DELETE /data/news/:id` → `{ ok: true }`
- `PUT /data/author` — `{ name?, instagram?, website?, description?, phone? }` → `{ ok: true }`
- `POST /push/register-token` — `{ token, platform: 'ios'|'android' }` → `{ ok: true }`
- `POST /push/unregister-token` — `{ token }` → `{ ok: true }`
- `GET /push/settings` → `{ news, tournaments, payments, schedule }` (все boolean)
- `PUT /push/settings` → body same, upsert
- `POST /upload` — `multipart/form-data` field `file` → `{ url: string }` (только авторизованным, max 10MB, image/*)

## 6. ОБЩИЕ КОМПОНЕНТЫ (src/components/)

### 6.1. GlassCard — главная карточка

Props: `children`, `style?`, `onPress?`, `padding?` (16/20/24), `radius?` (16/20/24/28).

Реализация: `View` + условный `Pressable` с press-scale анимацией (Reanimated `useSharedValue`), фон полупрозрачный с cross-платформ blur через `expo-blur` (`BlurView intensity={30} tint={dark ? 'dark' : 'light'}`). Border 1px, цвета из §3.1.

### 6.2. Avatar

Props: `src?: string | null`, `name: string`, `size?: number = 40`, `style?`.

Реализация:
- Если `src` — `<Image source={{ uri: src }} contentFit="cover" cachePolicy="memory-disk" />` (из expo-image).
- Если нет — круглый `View` с `bg: #dc2626`, по центру текст (первые 2 буквы имени, uppercase, fontSize = size * 0.35).

### 6.3. BottomSheet (модалка)

Props: `visible`, `onClose`, `title?`, `children`, `snapPoint?`.

Реализация: Reanimated slide up `translateY 100% → 0` за `0.4s cubic-bezier(0.22, 1, 0.36, 1)`. Backdrop `bg: rgba(0,0,0,0.6)` с fade 0.3s. Handle bar: `View 40x4 rounded-full` сверху. Close button справа сверху. `max-h: 85%`. SafeAreaBottom + 20px внутри.

### 6.4. PageHeader

Props: `title: string`, `back?: boolean`, `actions?: ReactNode`, `gradient?: boolean` (для логотипа).

Реализация: sticky top (`z-index`). При scroll > 10 — backgroundColor меняется от прозрачного до полупрозрачного (`bg-black/55` dark / `bg-white/55` light) + blur (через expo-blur). Back — `ChevronLeft` 24px. Title — 18px bold uppercase italic.

### 6.5. PhoneInput

Форматирование: `8 (900) 123-45-67`. Input `keyboardType="phone-pad"`. Внутри преобразует вход `7XXXXXXXXXX` → `8XXXXXXXXXX`, хранит в state только цифры, но показывает форматированно.

### 6.6. DateButton

Компонент-пилюля: `Calendar` icon 12px + текст даты. На tap открывает `DateTimePicker` из `@react-native-community/datetimepicker` (установить). Цвета: выбранная — `bg-red-500/15 text-red-500 border-red-500/30`, пустая — glass styling.

### 6.7. StatusBadge

Props: `status: 'sick' | 'injury' | 'skip' | null`. Маппинг:
- `sick`: Thermometer 10px + "Болеет", `bg-yellow-500/15 text-yellow-400`
- `injury`: HeartCrack 10px + "Травма", `bg-red-500/15 text-red-400`
- `skip`: Zap 10px + "Сачок", `bg-purple-500/15 text-purple-400`
- `null`: не рендерить

Сам бейдж: `px-2 py-1 rounded-full`, `flex-row gap-1`, `text-[10px] font-semibold uppercase`.

### 6.8. SportBadge

Props: `sport: string`. Показывает эмодзи + текст:
- bjj: 🥋 БЖЖ
- mma: 🥊 ММА
- boxing: 🥊 Бокс
- wrestling: 🤼 Вольная
- judo: 🥋 Дзюдо
- karate: 🥋 Карате
- kickboxing: 🥊 Кикбоксинг
- muaythai: 🥊 Муай-тай
- grappling: 🤼 Грэпплинг
- other: ⚡ Другое

Пилюля: `px-2 py-1 rounded-full bg-red-500/10 text-red-400 text-xs`.

### 6.9. BottomNav (tab bar)

5 табов зависят от роли (см. §4.4). Каждый таб:
- Icon 22px (stroke 2.5 active, 1.5 inactive)
- Label: 10px bold uppercase (опционально, видно на active)
- Active: bg `rgba(255,255,255,0.12)` (dark) / `rgba(0,0,0,0.06)` (light), `rounded-full` индикатор
- Контейнер: `rounded-[22px]`, `h: 60`, `mx: 16`, `mb: safeAreaBottom`, `bg: rgba(255,255,255,0.08)` dark / `rgba(255,255,255,0.5)` light, `backdrop-blur-xl`

### 6.10. PressableScale

Обёртка поверх `<Pressable>`: `onPressIn` → `scale(0.96) opacity(0.85)` за 120ms (Reanimated spring), `onPressOut` → обратно. Использовать везде где веб использует `press-scale`.

### 6.11. Тема (useTheme)

Zustand store `useThemeStore` с `{ dark: boolean, toggle() }`, персистит в SecureStore (ключ `iborcuha_theme`). По умолчанию: `dark = true`. Применяется ко всем компонентам через контекст (либо простой hook с подпиской на store).

## 8. PUSH-УВЕДОМЛЕНИЯ

### Регистрация

```typescript
// src/lib/pushNotifications.ts
import * as Device from 'expo-device';
import * as Notifications from 'expo-notifications';
import { api } from './apiClient';

Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowAlert: true,
    shouldShowBanner: true,
    shouldShowList: true,
    shouldPlaySound: true,
    shouldSetBadge: false,
  }),
});

export async function registerForPushNotifications() {
  if (!Device.isDevice) return null;
  const { status } = await Notifications.getPermissionsAsync();
  let final = status;
  if (status !== 'granted') {
    const res = await Notifications.requestPermissionsAsync();
    final = res.status;
  }
  if (final !== 'granted') return null;

  const projectId = Constants.expoConfig?.extra?.eas?.projectId;
  const token = (await Notifications.getExpoPushTokenAsync({ projectId })).data;
  await api.post('/push/register-token', {
    token,
    platform: Platform.OS === 'ios' ? 'ios' : 'android',
  });
  return token;
}

export function subscribeToNotificationTaps(navigate: (url: string) => void) {
  const sub = Notifications.addNotificationResponseReceivedListener((response) => {
    const url = response.notification.request.content.data?.url;
    if (typeof url === 'string') navigate(url);
  });
  return () => sub.remove();
}
```

Вызов `registerForPushNotifications()` после успешного login. Отписывать таки нужно при logout: `POST /push/unregister-token`.

## 10. APICLIENT (src/lib/apiClient.ts)

```typescript
import * as SecureStore from 'expo-secure-store';
import Constants from 'expo-constants';

const BASE = (Constants.expoConfig?.extra?.apiBaseUrl ?? 'https://iborcuha.ru') + '/api';
const TOKEN_KEY = 'iborcuha_token';

export async function getToken() { return SecureStore.getItemAsync(TOKEN_KEY); }
export async function setToken(t: string | null) {
  if (t) await SecureStore.setItemAsync(TOKEN_KEY, t);
  else await SecureStore.deleteItemAsync(TOKEN_KEY).catch(() => undefined);
}

export class ApiError extends Error {
  constructor(public status: number, public errorType: string | undefined, message: string) {
    super(message);
  }
}

let refreshInFlight: Promise<string | null> | null = null;
async function refreshToken(): Promise<string | null> {
  if (refreshInFlight) return refreshInFlight;
  refreshInFlight = (async () => {
    const t = await getToken();
    if (!t) return null;
    const r = await fetch(`${BASE}/auth/refresh`, {
      method: 'POST',
      headers: { Authorization: `Bearer ${t}` },
    });
    if (!r.ok) return null;
    const data = await r.json();
    if (data.token) { await setToken(data.token); return data.token; }
    return null;
  })().finally(() => { refreshInFlight = null; });
  return refreshInFlight;
}

async function request<T>(path: string, init: RequestInit = {}, retry = true): Promise<T> {
  const token = await getToken();
  const headers: Record<string, string> = {
    Accept: 'application/json',
    ...((init.body && !(init.body instanceof FormData)) ? { 'Content-Type': 'application/json' } : {}),
    ...(token ? { Authorization: `Bearer ${token}` } : {}),
  };
  const res = await fetch(`${BASE}${path}`, { ...init, headers });
  if (res.status === 401 && retry) {
    const fresh = await refreshToken();
    if (fresh) return request<T>(path, init, false);
    await setToken(null);
  }
  const text = await res.text();
  const data = text ? JSON.parse(text) : null;
  if (!res.ok) throw new ApiError(res.status, data?.errorType, data?.error ?? 'Request failed');
  return data as T;
}

export const api = {
  get: <T,>(p: string) => request<T>(p),
  post: <T,>(p: string, body?: unknown) => request<T>(p, { method: 'POST', body: body instanceof FormData ? body : JSON.stringify(body ?? {}) }),
  put:  <T,>(p: string, body?: unknown) => request<T>(p, { method: 'PUT',  body: JSON.stringify(body ?? {}) }),
  del:  <T,>(p: string, body?: unknown) => request<T>(p, { method: 'DELETE', body: body ? JSON.stringify(body) : undefined }),
  login: (phone: string, password: string) => request<any>('/auth/login', { method: 'POST', body: JSON.stringify({ phone, password }) }),
  me: () => request<any>('/auth/me'),
  logout: () => request<any>('/auth/logout', { method: 'POST' }),
  uploadImage: async (uri: string) => {
    const form = new FormData();
    form.append('file', { uri, name: 'photo.jpg', type: 'image/jpeg' } as any);
    return request<{ url: string }>('/upload', { method: 'POST', body: form });
  },
};
```

## 14. ЧЕГО НЕ ДЕЛАТЬ

- **Не менять бэкенд** — он продакшн, работает, контракт зафиксирован в OpenAPI
- **Не делать admin-only экраны** (AddTrainer, AddTournament, CreateInternalTournament, ClubDetail, Author edit) — они остаются в вебе
- **Не использовать** `fetch` напрямую в экранах — только через `api` клиент с refresh-логикой
- **Не использовать** `react-native` Image — только `expo-image`
- **Не использовать** `FlatList` для больших списков — только `FlashList`
- **Не делать** новые роли или права — только `superadmin | trainer | student` как в бэке
- **Не писать** комментарии без необходимости (только где «почему», не «что»)
- **Не использовать** `async storage` для токена — только SecureStore (ключи безопаснее)
